1. What WH2 Radio Is

WH2 Radio is a digital radio and music discovery system for AI-assisted creators and independent artists. The system allows creators to submit music, the owner/team to review submissions, approve music, organize the music library, manage artists, update station settings, and connect a Live365 radio player when ready.

WH2 stands for We Human Too. The station message is: AI-powered music discovery. Human approved.

2. Main Tools Used

GitHub

GitHub stores the website files. Every page on the WH2 site is a file inside the GitHub repository.

  • index.html = public homepage
  • submit.html = music submission form
  • login.html = owner login
  • admin.html = owner dashboard
  • submissions.html = review queue
  • library.html = approved music library
  • artists.html = artist database
  • instructions.html = this manual
  • terms.html, privacy.html, copyright.html, submission.html = legal pages

Firebase

Firebase controls login and stores data.

  • Authentication controls owner login.
  • Firestore Database stores submissions, approved songs, artist data, and station settings.

Live365

Live365 will be used for the actual radio player/stream when Troy provides the station link or embed code.

3. Daily Operating Routine

  1. Open the public WH2 homepage and make sure it loads.
  2. Login through login.html.
  3. Open the owner dashboard.
  4. Check Music Submissions.
  5. Review new songs carefully.
  6. Approve songs that are acceptable.
  7. Reject songs that are incomplete, suspicious, copyrighted, or not ready.
  8. Open Music Library and confirm approved songs appeared there.
  9. Open Artist Database and confirm the artist appears there.
  10. Update Featured Creator if needed.
  11. Update Station Message if needed.
  12. Verify Live365 player is working once connected.

4. How to Edit the Website in GitHub

  1. Go to GitHub.
  2. Open the WH2 repository.
  3. Click the file you want to edit.
  4. Click the pencil icon.
  5. If replacing the full page, press Ctrl + A and delete everything.
  6. Paste the new full code.
  7. Scroll down to Commit Changes.
  8. Write a clear commit message like Update WH2 homepage.
  9. Click Commit Changes.
  10. Wait 1–3 minutes, then refresh the website.
Do not randomly delete files. Do not rename files unless you know every button linking to that file has also been updated.

5. What to Ask ChatGPT When Editing Code

If someone is using free ChatGPT or another AI tool, they need to give clear instructions and paste the current code when possible.

Best Prompt for Code Edits

I am editing a GitHub Pages website for WH2 Radio. I need a full fresh HTML code replacement for [file name]. Keep the Firebase config the same. Keep the WH2 red and black branding. Do not remove working features. Here is the current code: [paste code].

Best Prompt for Fixing Broken Buttons

This button is supposed to open [page name], but it does not work. Review the file names and give me the full corrected code.

Best Prompt for Adding Features

Add this feature without breaking the current system. Provide full code, exact file name, commit message, and testing steps.

Never ask AI to “just fix it” without telling it the file name, current problem, and what should happen.

6. Submission Review Process

  1. Go to the dashboard.
  2. Click Open Submissions.
  3. Review artist/creator name.
  4. Review email.
  5. Review song title.
  6. Review submission type.
  7. Review genre.
  8. Open song link if provided.
  9. Open cover link if provided.
  10. Read notes.
  11. Confirm checkboxes were completed.
  12. If acceptable, click Approve + Add to Library.
  13. If not acceptable, click Reject Submission.
Do not approve copyrighted commercial songs, stolen beats, unauthorized samples, fake links, or anything that looks legally risky.

7. Music Library Process

The Music Library stores approved songs only.

  1. Go to dashboard.
  2. Click Open Library.
  3. Confirm approved song appears.
  4. Review artist name, genre, links, and status.
  5. If a song should not remain approved, delete it from the library.
  6. If a creator should be highlighted, click Set as Featured Creator.

8. Artist Database Process

The Artist Database is created from approved songs.

  1. Go to dashboard.
  2. Click Open Artists.
  3. Find the artist/creator.
  4. Review approved songs and genre tags.
  5. Use this page to see who is active inside WH2.
  6. Set featured creator from this page if needed.

9. Featured Creator Process

  1. Open dashboard.
  2. Find Featured Creator section.
  3. Enter creator name.
  4. Enter short bio.
  5. Add image link if available.
  6. Add creator/music/social link if available.
  7. Click Save Featured Creator.
  8. Open public site and confirm it looks correct.

10. Station Message Process

  1. Open dashboard.
  2. Find Station Message.
  3. Type the message for the public homepage.
  4. Click Save Message.
  5. Open homepage and verify the message changed.

Good station message examples:

  • WH2 Radio highlights AI-powered music discovery and independent sound.
  • Welcome to WH2 Radio — We Human Too.
  • AI-powered music. Human approved.

11. Live365 Process

Live365 is expected to handle the real radio streaming side.

  1. Get the Live365 station link or embed code.
  2. Login to WH2 dashboard.
  3. Paste Live365 info into the Live365 section.
  4. Click Save Live365 Info.
  5. Developer must connect that saved data to the homepage player area.
  6. Test on phone and computer.

12. Copyright Complaint Process

  1. Take complaint seriously.
  2. Write down who complained.
  3. Write down song title and artist name.
  4. Go to Music Library.
  5. Remove or hide the song while reviewing.
  6. Check submission details.
  7. Ask creator for proof of rights if needed.
  8. If rights cannot be confirmed, keep the song removed.

13. Emergency Procedures

  • Website not loading: Check GitHub Pages deployment.
  • Login not working: Check Firebase Authentication.
  • Submissions not saving: Check Firebase Firestore rules and config.
  • Buttons not opening: Confirm the target file exists with the exact same name.
  • Live365 not playing: Check Live365 station status and embed code.
  • Bad song approved: Delete from library immediately.
  • Legal complaint: Remove content first, investigate second.

14. Training Someone New

Before someone helps run WH2, they must understand:

  • How to login
  • How to review submissions
  • How to approve/reject music
  • How to check the library
  • How to update featured creator
  • How to update station message
  • How to avoid copyright problems
  • How to ask ChatGPT for safe code edits
